**版本**: v0.1.0

## 一、环境要求

### 硬件要求

| 环境 | CPU | 内存 | 磁盘 |
|------|-----|------|------|
| 开发环境 | 2 核 | 4GB | 20GB |
| 生产环境 | 8 核 | 16GB | 100GB |

### 软件要求

- Python 3.11+
- PostgreSQL 14+ / SQLite 3.35+
- Redis 5.0+
- Nginx 1.20+ (可选)
- Docker 20.10+ (可选)

## 二、开发环境快速部署

### 1. 克隆项目并安装依赖

```bash
git clone <repository-url>
cd ev_check

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，配置数据库等信息
```

**开发环境 .env 示例**:

```bash
DEBUG=True
LOG_LEVEL=DEBUG
SECRET_KEY=dev-secret-key
DATABASE_URL=sqlite:///./ev_check.db
REDIS_URL=redis://localhost:6379/0
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

### 3. 初始化数据库

```bash
# 创建数据库表
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"

# 创建管理员账号
python scripts/create_admin.py
```

### 4. 启动服务

```bash
# 终端 1: 启动 Web 服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 终端 2: 启动 Celery Worker
celery -A app.celery_config worker --loglevel=info

# 终端 3: 启动 Celery Beat
celery -A app.celery_config beat --loglevel=info
```

### 5. 访问系统

- Web 界面: http://localhost:8000
- API 文档: http://localhost:8000/docs
- 默认账号: admin / admin123

## 三、Docker 快速部署

### 1. 使用 Docker Compose

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 初始化数据库
docker-compose exec web alembic upgrade head
docker-compose exec web python scripts/create_admin.py
```

### 2. 访问系统

- Web 界面: http://localhost:8000
- 默认账号: admin / admin123

## 四、生产环境部署要点

### 1. 系统准备

```bash
# 安装系统依赖
sudo apt update && sudo apt install -y \
    python3.11 python3.11-venv \
    postgresql redis-server nginx

# 创建应用用户
sudo useradd -m -s /bin/bash evcheck
```

### 2. 配置 PostgreSQL

```bash
sudo su - postgres
psql
CREATE DATABASE ev_check;
CREATE USER evcheck WITH PASSWORD 'secure-password';
GRANT ALL PRIVILEGES ON DATABASE ev_check TO evcheck;
\q
exit
```

### 3. 部署应用

```bash
sudo su - evcheck
cd /home/evcheck
git clone <repository-url> ev_check
cd ev_check

python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 4. 配置生产环境变量

```bash
vim .env
```

```bash
DEBUG=False
LOG_LEVEL=INFO
SECRET_KEY=your-very-secure-secret-key
DATABASE_URL=postgresql+asyncpg://evcheck:secure-password@localhost:5432/ev_check
REDIS_URL=redis://localhost:6379/0
```

### 5. 配置 Systemd 服务

创建服务文件:
- `/etc/systemd/system/evcheck-web.service`
- `/etc/systemd/system/evcheck-worker.service`
- `/etc/systemd/system/evcheck-beat.service`

```bash
sudo systemctl daemon-reload
sudo systemctl start evcheck-web evcheck-worker evcheck-beat
sudo systemctl enable evcheck-web evcheck-worker evcheck-beat
```

### 6. 配置 Nginx

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /static/ {
        alias /home/evcheck/ev_check/app/static/;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/evcheck /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 六、常见问题

### 1. 数据库连接失败

```bash
# 检查数据库服务
sudo systemctl status postgresql

# 测试连接
psql -h localhost -U evcheck -d ev_check
```

### 2. Redis 连接失败

```bash
# 检查 Redis 服务
sudo systemctl status redis

# 测试连接
redis-cli ping
```

### 3. Celery Worker 不工作

```bash
# 查看日志
sudo journalctl -u evcheck-worker -f

# 重启服务
sudo systemctl restart evcheck-worker
```

## 七、备份与恢复

### 数据库备份

```bash
# 备份
pg_dump -h localhost -U evcheck ev_check > backup_$(date +%Y%m%d).sql

# 恢复
psql -h localhost -U evcheck ev_check < backup_20260313.sql
```

### 自动备份

```bash
# 添加到 crontab
0 2 * * * pg_dump -h localhost -U evcheck ev_check | gzip > /backups/db_$(date +\%Y\%m\%d).sql.gz
```

## 八、安全建议

1. **修改默认密码**: 首次登录后立即修改管理员密码
2. **使用 HTTPS**: 配置 SSL 证书 (Let's Encrypt)
3. **防火墙配置**: 只开放必要端口 (80, 443)
4. **定期备份**: 配置自动备份任务
5. **更新依赖**: 定期更新系统和 Python 包

## 九、性能优化

1. **数据库索引**: 为常用查询字段添加索引

3. **Nginx 缓存**: 配置静态文件缓存
