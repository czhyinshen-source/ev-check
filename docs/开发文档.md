# 运行环境检查系统 - 开发文档

## 1. 系统概述

### 1.1 系统背景
为确保通信系统运行环境的正确性，替代人工检查方式，提高检查效率和质量，防止程序、配置等信息被有意或无意改动，特开发本运行环境检查系统。

### 1.2 核心目标
- 自动化检查通信机运行环境
- 提供快照对比机制
- 支持手动和定时检查
- 生成检查报告
- 支持多操作员并发访问

## 2. 核心概念

### 2.3 实体关系

```
通信机分组 (1) ──< 通信机 (1..*)
通信机 (1) ──< 运行环境 (*)
运行环境 (*) >── (1) 快照
快照组 (1) ──< 快照 (*)
快照 (1) ──< 检查项列表 (*)
检查项列表 (1) ──< 检查项 (*)
检查规则 (1) ──< 检查项列表 (0..1)
检查规则 (1) ──< 快照 (*)
```

### 2.4 核心定义

| 概念 | 说明 | 示例 |
|------|------|------|
| **通信机** | 受系统管理、控制的服务器 | 名称、地址、端口 |
| **运行环境** | 应用程序依赖的所有环境 | 程序、配置、系统参数、服务、网络 |
| **检查项** | 最小检查单元 | 文件修改时间、文件大小、MD5 值 |
| **检查项列表** | 检查项的集合 | 定义需要检查的内容 |
| **快照组** | 包含检查项列表，定义快照应包含的项目，支持树形结构 | 系统 A/生产环境、系统 B/测试环境 |
| **快照** | 一台或多台通信机的运行环境状态 | 快照时间、快照实例 |
| **检查规则** | 由检查项列表和快照组成，决定检查逻辑 | 检查哪些项目、与什么数据比对 |

**快照分组层次结构**:
- 支持多层级树形结构
- 每个分组可包含子分组和快照
- 顶级分组称为根分组
- 分组可关联检查项列表
- 支持系统分组 (不可删除)

## 5. 技术架构

### 5.1 技术栈选择
根据项目规则，优先使用 Python 及相关框架：

**后端**:
- Web 框架：FastAPI (高性能、异步支持)
- 数据库：SQLite (轻量级) / PostgreSQL (生产环境)
- ORM：SQLAlchemy
- 任务队列：Celery + Redis (定时任务、异步检查)
- 缓存：Redis

**前端**:
- 框架：Vue.js 3 (响应式、易上手)
- UI 组件库：Element Plus (定制化主题)
- 状态管理：Pinia
- 图表：ECharts (报表展示)
- HTTP 客户端：Axios
- CSS 框架：自定义 (基于深交所设计系统)

**部署**:
- Web 服务器：Nginx + Gunicorn/Uvicorn
- 容器化：Docker
- 监控：Prometheus + Grafana

```python
# 伪代码示例
class CheckEngine:
    async def execute_check(self, check_rule: CheckRule) -> CheckResult:
        """执行环境检查"""
        # 1. 获取检查项列表
        check_items = await self.get_check_items(check_rule)
        
        # 2. 获取目标通信机
        communications = await self.get_communications(check_rule)
        
        # 3. 获取基准快照
        baseline_snapshot = await self.get_baseline_snapshot(check_rule)
        
        # 4. 并行执行检查
        tasks = []
        for comm in communications:
            task = self.check_communication(comm, check_items, baseline_snapshot)
            tasks.append(task)
        
        results = await asyncio.gather(*tasks)
        return CheckResult(results=results)
    
    async def check_communication(self, comm, check_items, baseline):
        """检查单台通信机"""
        async with SSHClient(comm) as client:
            for item in check_items:
                # 收集当前数据
                current_data = await self.collect_data(client, item)
                # 获取基准数据
                baseline_data = await self.get_baseline_data(baseline, item)
                # 比较数据
                status = self.compare(current_data, baseline_data)
                yield CheckResultDetail(item=item, status=status)
```

### 8.2 快照服务

```python
class SnapshotService:
    """快照管理服务"""
    
    async def create_snapshot(self, snapshot_data: SnapshotCreate) -> Snapshot:
        """创建快照"""
        # 1. 验证快照名称唯一性
        await self.validate_name_uniqueness(
            snapshot_data.name, 
            snapshot_data.group_id
        )
        
        # 2. 创建快照记录
        snapshot = Snapshot(
            name=snapshot_data.name,
            group_id=snapshot_data.group_id,
            snapshot_time=datetime.now(),
            created_by=current_user.username,
            is_default=snapshot_data.is_default,
            description=snapshot_data.description
        )
        
        # 3. 为每个通信机创建快照实例
        for comm_id in snapshot_data.communication_ids:
            instance = SnapshotInstance(
                snapshot_id=snapshot.id,
                communication_id=comm_id,
                check_item_list_id=snapshot_data.check_item_list_id
            )
            await self.db.add(instance)
        
        # 4. 异步构建快照数据
        await self.build_snapshot_data(snapshot.id)
        
        return snapshot
    
    async def build_snapshot_data(self, snapshot_id: int):
        """构建快照数据"""
        # 1. 获取快照实例列表
        instances = await self.get_snapshot_instances(snapshot_id)
        
        # 2. 并行收集各通信机的环境数据
        tasks = []
        for instance in instances:
            task = self.collect_environment_data(instance)
            tasks.append(task)
        
        await asyncio.gather(*tasks)
    
    async def collect_environment_data(self, instance: SnapshotInstance):
        """收集单个通信机的环境数据"""
        async with SSHClient(instance.communication) as client:
            # 收集检查项数据
            for check_item in instance.check_items:
                data = await self.collect_check_item_data(client, check_item)
                await self.save_environment_data(instance.id, check_item.id, data)
    
    async def get_snapshot_content(self, snapshot_id: int) -> SnapshotContent:
        """获取快照内容详情"""
        # 1. 获取快照基本信息
        snapshot = await self.get_snapshot(snapshot_id)
        
        # 2. 获取所有快照实例
        instances = await self.get_snapshot_instances(snapshot_id)
        
        # 3. 按通信机组织内容
        content = {}
        for instance in instances:
            comm_data = await self.get_communication_snapshot_data(instance)
            content[instance.communication.name] = comm_data
        
        return SnapshotContent(
            snapshot=snapshot,
            communications=content
        )
    
    async def get_communication_snapshot_data(
        self, 
        instance: SnapshotInstance
    ) -> CommunicationSnapshotData:
        """获取单个通信机的快照数据"""
        # 1. 获取所有环境数据
        env_data_list = await self.get_environment_data(instance.id)
        
        # 2. 组织文件内容
        files = {}
        system_configs = {}
        
        for env_data in env_data_list:
            check_item = env_data.check_item
            if check_item.type == "file":
                files[check_item.path] = env_data.data_value
            elif check_item.type == "system_config":
                system_configs[check_item.name] = env_data.data_value
        
        return CommunicationSnapshotData(
            communication=instance.communication,
            files=files,
            system_configs=system_configs
        )
```

```python
            name=group_data.name,
            parent_id=group_data.parent_id,
            check_item_list_id=group_data.check_item_list_id,
            is_system=group_data.is_system,
            description=group_data.description
        )
        
        await self.db.add(group)
        return group
    
    async def get_group_tree(self) -> List[GroupTreeNode]:
        """获取快照组树形结构"""
        # 1. 获取所有分组
        groups = await self.get_all_groups()
        
        # 2. 构建树形结构
        tree = []
        group_map = {g.id: g for g in groups}
        
        for group in groups:
            if group.parent_id is None:
                tree.append(self.build_tree_node(group, group_map))
        
        return tree
    
    def build_tree_node(
        self, 
        group: SnapshotGroup, 
        group_map: Dict[int, SnapshotGroup]
    ) -> GroupTreeNode:
        """递归构建树节点"""
        children = [
            self.build_tree_node(g, group_map) 
            for g in group_map.values() 
            if g.parent_id == group.id
        ]
        
        return GroupTreeNode(
            group=group,
            children=children,
            snapshot_count=await self.count_snapshots(group.id)
        )
    
    async def delete_group(self, group_id: int):
        """删除快照分组"""
        group = await self.get_group(group_id)
        
        # 1. 检查是否为系统分组
        if group.is_system:
            raise SystemGroupError("系统分组不可删除")
        
        # 2. 检查是否有子分组
        children = await self.get_child_groups(group_id)
        if children:
            raise GroupHasChildrenError("请先删除子分组")
        
        # 3. 检查是否有快照
        snapshots = await self.get_snapshots(group_id)
        if snapshots:
            raise GroupHasSnapshotsError("请先删除或迁移快照")
        
        # 4. 删除分组
        await self.db.delete(group)
```

```python
class CheckItemValidator:
    """检查项验证器"""
    
    @staticmethod
    def validate_file_path(path: str) -> bool:
        """验证文件路径格式"""
        if not path or not path.startswith('/'):
            return False
        # 检查路径合法性
        return True
    
    @staticmethod
    def validate_port(port: int) -> bool:
        """验证端口号"""
        return 1 <= port <= 65535
    
    @staticmethod
    def validate_md5(md5_hash: str) -> bool:
        """验证 MD5 格式"""
        if not md5_hash or len(md5_hash) != 32:
            return False
        return all(c in '0123456789abcdef' for c in md5_hash.lower())
    
    @staticmethod
    def validate_permission_octal(octal: str) -> bool:
        """验证权限八进制格式"""
        if not octal or len(octal) not in [3, 4]:
            return False
        return all(c in '01234567' for c in octal)
    
    @staticmethod
    def validate_cidr(cidr: str) -> bool:
        """验证 CIDR 格式"""
        try:
            ipaddress.ip_network(cidr, strict=False)
            return True
        except ValueError:
            return False
    
    async def validate_check_item(self, item_data: CheckItemCreate) -> List[str]:
        """验证检查项数据"""
        errors = []
        
        # 验证路径
        if item_data.target_path and not self.validate_file_path(item_data.target_path):
            errors.append("文件路径格式不正确")
        
        # 验证检查属性
        attrs = item_data.check_attributes
        
        if attrs.get('port', {}).get('enabled'):
            port = attrs['port'].get('port_number')
            if not self.validate_port(port):
                errors.append("端口号范围为 1-65535")
        
        if attrs.get('md5', {}).get('enabled'):
            md5 = attrs['md5'].get('hash')
            if not self.validate_md5(md5):
                errors.append("MD5 格式不正确")
        
        if attrs.get('permission', {}).get('enabled'):
            perm = attrs['permission'].get('octal')
            if not self.validate_permission_octal(perm):
                errors.append("权限格式不正确")
        
        return errors
```

```python
        passphrase: str = None
    ) -> SSHKey:
        """生成 SSH 密钥对"""
        # 1. 生成密钥对
        if key_type == 'rsa':
            private_key = rsa.generate_private_key(
                public_exponent=65537,
                key_size=key_bits,
                backend=default_backend()
            )
        elif key_type == 'ecdsa':
            private_key = ec.generate_private_key(
                ec.SECP521R1(),
                backend=default_backend()
            )
        elif key_type == 'ed25519':
            private_key = ed25519.Ed25519PrivateKey.generate()
        else:
            raise ValueError(f"不支持的密钥类型：{key_type}")
        
        # 2. 序列化私钥
        private_key_pem = private_key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.OpenSSH,
            encryption_algorithm=serialization.BestAvailableEncryption(passphrase.encode()) if passphrase else serialization.NoEncryption()
        )

            encoding=serialization.Encoding.OpenSSH,
            format=serialization.PublicFormat.OpenSSH
        )
        
        # 4. 计算指纹
        fingerprint = self.calculate_fingerprint(public_key_pem)

    def calculate_fingerprint(self, public_key: str) -> str:
        """计算密钥指纹"""
        try:

        except Exception:
            return None
    
    def detect_key_type(self, public_key: str) -> str:
        """检测密钥类型"""
        if 'ssh-rsa' in public_key:
            return 'rsa'
        elif 'ssh-ec' in public_key or 'ecdsa' in public_key:
            return 'ecdsa'
        elif 'ssh-ed25519' in public_key:
            return 'ed25519'
        return 'unknown'
```

## 9. 定时任务

使用 Celery Beat 实现定时检查：

```python
# Celery 配置
from celery import Celery
from celery.schedules import crontab

app = Celery('ev_check', broker='redis://localhost:6379/0')

app.conf.beat_schedule = {
    'daily-check': {
        'task': 'app.tasks.scheduled_tasks.run_scheduled_check',
        'schedule': crontab(hour=2, minute=0),  # 每天凌晨 2 点
    },
    'communication-status-check': {
        'task': 'app.tasks.scheduled_tasks.check_communication_status',
        'schedule': 60.0,  # 每分钟执行一次
    },
}
```

**通信机状态检查任务**:
- 每分钟自动测试所有通信机的连接状态
- 更新通信机的状态（在线/离线/未连接）
- 状态判断逻辑：
  - 未测试过连通性：未连接
  - 测试成功：在线
  - 测试失败：离线
- 状态修改依赖于上一次测试连通性的结果

**对话框元素**:

  - 通信机名称
  - IP 地址
  - 端口号
  - 用户名
- 表单字段:
  - **选择 SSH 密钥** (必填)
    - 下拉选择框
    - 显示可用的 SSH 密钥列表
    - 支持搜索过滤
  - **部署方式** (必填)
    - 单选按钮组
    - 选项：自动部署 / 手动部署
  - **当前密码** (条件必填)
    - 密码输入框
    - 自动部署时必填
    - 用于首次连接通信机

  - 取消按钮
  - 帮助按钮

1. 选择 SSH 密钥
2. 输入当前密码
3. 点击"部署公钥"按钮
4. 系统使用密码连接通信机
5. 自动上传公钥到通信机 ~/.ssh/authorized_keys
6. 验证免密登录是否成功
7. 部署成功后保存配置

**手动部署说明**:
1. 选择或生成 SSH 密钥
2. 复制公钥内容
3. 手动登录通信机
4. 将公钥添加到 ~/.ssh/authorized_keys
5. 设置正确的权限 (600)
6. 回到系统点击"验证"按钮

**交互流程**:
1. 点击通信机列表中的"配置 SSH"按钮
2. 选择要使用的 SSH 密钥

4. 自动部署时输入当前密码
5. 点击"部署公钥"按钮

7. 显示部署进度
8. 部署成功后保存配置
9. 关闭对话框

**错误处理**:
- 连接失败时，显示"无法连接到通信机，请检查网络和密码"
- 公钥部署失败时，显示具体错误信息
- 权限不足时，提示"需要 sudo 权限来配置 authorized_keys"
- 验证失败时，提示"免密登录验证失败，请检查配置"
