# 登录用户和通信机信息显示问题修复说明

## 问题分析

### 1. 登录API请求格式问题
**问题描述：**
前端使用 `FormData` 发送登录请求，但后端期望 `OAuth2PasswordRequestForm` 格式（`application/x-www-form-urlencoded`）。

**修复方案：**
将 `FormData` 改为 `URLSearchParams`，并设置正确的 `Content-Type` 头。

**修改文件：** `app/static/login.html`

**修改前：**
```javascript
const formData = new FormData();
formData.append('username', username);
formData.append('password', password);

const response = await fetch('/api/v1/users/login', {
    method: 'POST',
    body: formData
});
```

**修改后：**
```javascript
const formData = new URLSearchParams();
formData.append('username', username);
formData.append('password', password);

const response = await fetch('/api/v1/users/login', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: formData
});
```

### 2. 用户信息显示问题
**问题描述：**
用户信息可能显示为"未知"，且缺少错误处理。

**修复方案：**
增加错误处理和调试日志。

**修改文件：** `app/static/js/dashboard.js`

**修改前：**
```javascript
if (!token) window.location.href = '/login.html';
document.getElementById('currentUser').textContent = '用户: ' + (localStorage.getItem('username') || '未知');
```

**修改后：**
```javascript
if (!token) {
    console.error('未找到token，重定向到登录页面');
    window.location.href = '/login.html';
}

const username = localStorage.getItem('username');
const currentUserEl = document.getElementById('currentUser');
if (currentUserEl) {
    currentUserEl.textContent = '用户: ' + (username || '未知');
    console.log('当前用户:', username);
} else {
    console.error('未找到currentUser元素');
}
```

### 3. 通信机列表加载问题
**问题描述：**
通信机列表加载失败时没有明确的错误提示。

**修复方案：**
增加详细的错误处理和日志记录。

**修改文件：** `app/static/js/communications.js`

**修改前：**
```javascript
async function loadCommunications() {
    const { API_BASE, getHeaders } = window.shared;
    try {
        const res = await fetch(`${API_BASE}/api/v1/communications`, { headers: getHeaders() });
        let comms = await res.json();
        // ... 处理数据
    } catch (e) { console.error(e); }
}
```

**修改后：**
```javascript
async function loadCommunications() {
    const { API_BASE, getHeaders } = window.shared;
    try {
        console.log('开始加载通信机列表...');
        const res = await fetch(`${API_BASE}/api/v1/communications`, { headers: getHeaders() });

        if (!res.ok) {
            throw new Error(`HTTP ${res.status}: ${res.statusText}`);
        }

        let comms = await res.json();
        console.log('成功加载通信机列表，数量:', comms.length);
        // ... 处理数据
    } catch (error) {
        console.error('加载通信机列表失败:', error);
        const tbody = document.getElementById('commTable');
        if (tbody) {
            tbody.innerHTML = '<tr><td colspan="6" class="empty-state">加载失败: ' + error.message + '</td></tr>';
        }
    }
}
```

## 常见问题排查

### 问题1: 登录后立即退出
**可能原因：**
- Token 无效或过期
- 后端认证失败

**解决方案：**
1. 检查浏览器控制台的错误信息
2. 检查网络请求的响应状态
3. 确认后端服务正常运行

### 问题2: 用户名显示为"未知"
**可能原因：**
- localStorage 中没有存储 username
- 登录时没有正确保存用户信息

**解决方案：**
1. 在控制台检查：`localStorage.getItem('username')`
2. 重新登录
3. 检查登录页面是否正确保存了 username

### 问题3: 通信机列表为空
**可能原因：**
- 数据库中没有通信机数据
- API 请求失败
- 权限不足

**解决方案：**
1. 检查网络请求状态
2. 确认数据库中有通信机数据
3. 检查用户权限
4. 查看控制台的错误信息

### 问题4: API 请求 401 错误
**可能原因：**
- Token 过期或无效
- 请求头格式错误

**解决方案：**
1. 重新登录获取新 token
2. 检查 `getHeaders()` 函数是否正确设置 Authorization 头
3. 确认 token 格式：`Bearer <token>`
