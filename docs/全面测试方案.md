## 一、测试策略概述

### 1.1 测试目标

本测试方案旨在确保运行环境检查系统完全按照开发文档 (`docs/开发文档.md`) 和 spec 文件 (`.trae/specs/ev-check-system/`) 的要求实现，保证需求与落地一一对应。

**核心目标**:
- ✅ 验证所有功能需求的实现
- ✅ 确保非功能需求达标
- ✅ 发现并跟踪缺陷
- ✅ 提供质量评估报告
- ✅ 确保系统可投入生产使用

### 1.2 测试原则

根据前端测试最佳实践，本方案遵循以下原则:

1. **优先 E2E 测试** - 测试完整用户流程而非孤立组件
2. **最小化 Mock** - 使用真实环境，避免复杂 Mock
3. **测试行为而非实现** - 关注用户可见的行为
4. **分层测试策略** - 单元测试 + 集成测试 + E2E 测试

### 1.3 测试范围

#### 包含的测试内容

| 测试类型 | 测试对象 | 优先级 |
|---------|---------|--------|
| 单元测试 | 纯函数、工具类、服务层 | P1 |
| 集成测试 | API 接口、数据库操作、SSH 连接 | P0 |
| E2E 测试 | 完整用户流程、前端界面 | P0 |
| 性能测试 | 响应时间、并发能力 | P1 |
| 安全测试 | 认证授权、数据加密 | P1 |

#### 不包含的测试内容

- ❌ 第三方库的内部逻辑 (如 FastAPI、SQLAlchemy)
- ❌ 操作系统级别的 SSH 协议实现

```python
# tests/performance/test_concurrent.py

import pytest
import asyncio
from httpx import AsyncClient
import time

class TestConcurrentAccess:
    """并发访问测试"""
    
    @pytest.mark.asyncio
    async def test_5_concurrent_users(self, base_url):
        """5 个并发用户测试"""
        users = [
            {"username": f"user{i}", "password": "Test@123"}
            for i in range(5)
        ]
        
        async def login_and_query(user_data):
            """登录并查询"""
            async with AsyncClient() as client:
                # 登录
                response = await client.post(f"{base_url}/api/v1/users/login", json=user_data)
                token = response.json()["access_token"]
                headers = {"Authorization": f"Bearer {token}"}
                
                # 查询通信机列表
                start = time.time()
                response = await client.get(f"{base_url}/api/v1/communications", headers=headers)
                elapsed = time.time() - start
                
                return {
                    "success": response.status_code == 200,
                    "elapsed": elapsed,
                    "status_code": response.status_code
                }
        
        # 并发执行
        tasks = [login_and_query(user) for user in users]
        results = await asyncio.gather(*tasks)
        
        # 验证结果
        for i, result in enumerate(results):
            assert result["success"], f"用户{i + 1}请求失败"
            assert result["elapsed"] < 5.0, f"用户{i + 1}响应时间超过 5 秒"
        
        # 统计
        avg_time = sum(r["elapsed"] for r in results) / len(results)
        print(f"平均响应时间：{avg_time:.2f}秒")
    
    @pytest.mark.asyncio
    async def test_concurrent_check_execution(self, base_url, auth_headers):
        """并发执行检查任务 (应只有一个成功)"""
        async def start_check():
            async with AsyncClient() as client:
                response = await client.post(
                    f"{base_url}/api/v1/checks",
                    json={"rule_id": 1},
                    headers=auth_headers
                )
                return response.status_code
        
        # 并发启动 5 个检查任务
        tasks = [start_check() for _ in range(5)]
        results = await asyncio.gather(*tasks)
        
        # 只有一个应该成功 (201), 其他应该失败 (409 冲突)
        success_count = sum(1 for r in results if r == 201)
        conflict_count = sum(1 for r in results if r == 409)
        
        assert success_count == 1, "应该只有一个检查任务成功启动"
        assert conflict_count == 4, "其他 4 个任务应该返回冲突"
```
