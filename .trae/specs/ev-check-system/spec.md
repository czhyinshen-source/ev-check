# 运行环境检查系统 - 规范大纲

## Why
为确保通信系统运行环境的正确性，替代人工检查方式，提高检查效率和质量，防止程序、配置等信息被有意或无意改动，特开发本运行环境检查系统。

## What Changes
- 构建完整的运行环境检查系统，包含后端 API 服务、数据库模型、定时任务处理
- 实现快照对比机制，支持手动和定时环境检查
- 提供检查项管理、快照管理、环境检查、报表生成等核心功能
- 支持多操作员并发访问
- 新增前端登录页面和控制台界面
- 新增通信机管理功能，支持 SSH 免密认证

## Impact
- Affected specs: 检查项管理、快照管理、环境检查、数据管理、报表功能、通信机管理、系统管理
- Affected code: 后端 FastAPI 应用、数据库模型、Celery 任务、前端界面

## ADDED Requirements

### Requirement: 会话管理
系统 SHALL 提供用户认证功能

#### Scenario: 用户登录
- **WHEN** 用户访问根路径 http://localhost:8000/
- **THEN** 显示登录页面，包含用户名、密码输入框和登录按钮
- **THEN** 登录页面包含注册、忘记密码、修改密码链接

#### Scenario: 登录成功跳转
- **WHEN** 用户输入正确的用户名密码并点击登录
- **THEN** 系统验证凭据，成功后自动跳转到控制台页面 /dashboard.html

#### Scenario: 用户登出
- **WHEN** 用户点击控制台页面的退出登录按钮
- **THEN** 清除本地存储的 Token 并跳转回登录页面

#### Scenario: 用户注册
- **WHEN** 用户点击登录页面的注册链接
- **THEN** 进入注册页面，显示用户名、密码、确认密码输入框
- **WHEN** 用户填写注册信息并点击注册按钮
- **THEN** 系统验证用户名唯一性和密码一致性
- **THEN** 验证通过后创建用户并跳转到登录页面

#### Scenario: 忘记密码
- **WHEN** 用户点击登录页面的忘记密码链接
- **THEN** 弹出提示对话框，提示联系管理员修改密码

#### Scenario: 修改密码
- **WHEN** 用户点击登录页面的修改密码链接
- **THEN** 进入修改密码页面，显示用户名、旧密码、新密码、确认密码输入框
- **WHEN** 用户填写修改密码信息并点击保存按钮
- **THEN** 系统验证用户名唯一性、旧密码正确性和新密码一致性
- **THEN** 验证通过后更新密码并跳转到登录页面

### Requirement: 控制台界面
系统 SHALL 提供 Web 控制台界面

#### Scenario: 控制台首页
- **WHEN** 用户登录成功后访问控制台
- **THEN** 显示统计概览：通信机数量、检查项数量、快照数量、最近检查状态

#### Scenario: 标签页切换
- **WHEN** 用户点击不同功能标签
- **THEN** 切换显示对应功能模块：通信机管理、检查项管理、快照管理、检查执行

### Requirement: 通信机管理
系统 SHALL 提供通信机的完整管理能力

#### Scenario: 添加通信机
- **WHEN** 用户点击"添加通信机"按钮
- **THEN** 显示添加对话框，包含通信机名称、IP 地址、端口号、用户名、认证方式等字段

#### Scenario: 编辑通信机
- **WHEN** 用户点击通信机的"编辑"按钮
- **THEN** 显示编辑对话框，预填当前通信机信息

#### Scenario: 删除通信机
- **WHEN** 用户点击通信机的"删除"按钮
- **THEN** 弹出确认对话框，确认后删除通信机

#### Scenario: 通信机分组管理
- **WHEN** 用户管理通信机分组
- **THEN** 支持树形结构显示分组，支持创建、编辑、删除分组

#### Scenario: SSH 免密认证配置
- **WHEN** 用户配置 SSH 免密认证
- **THEN** 支持选择 SSH 密钥、自动/手动部署公钥到通信机

#### Scenario: 测试通信机连接
- **WHEN** 用户点击"测试连接"按钮
- **THEN** 系统尝试 SSH 连接，返回连接成功/失败状态

#### Scenario: 通信机状态显示
- **WHEN** 界面显示通信机列表
- **THEN** 显示通信机状态，根据测试连接结果判断
  - 未测试过连通性：未连接（灰色）
  - 测试成功：在线（绿色）
  - 测试失败：离线（红色）

#### Scenario: 定时测试通信机连接
- **WHEN** 系统运行时
- **THEN** 后端服务每分钟自动测试所有通信机的连接状态
- **THEN** 自动更新通信机的状态显示
- **THEN** 状态修改依赖于上一次测试连通性的结果

#### Scenario: Excel导入通信机
- **WHEN** 用户点击"导入Excel"按钮
- **THEN** 弹出导入对话框，支持上传Excel文件
- **WHEN** 用户上传Excel文件并点击导入
- **THEN** 系统验证Excel文件格式和数据
- **THEN** 验证通过后批量创建通信机
- **THEN** 显示导入成功提示，包含导入成功的通信机数量

#### Scenario: 通信机分组处理
- **WHEN** 用户在Excel模板中填写通信机分组
- **THEN** 系统根据分组名称自动匹配对应的分组ID
- **THEN** 分组ID只在数据库后端可见

#### Scenario: 批量部署公钥
- **WHEN** 用户选择多个通信机并点击"批量部署公钥"按钮
- **THEN** 弹出部署对话框，支持选择SSH密钥
- **WHEN** 用户选择密钥并点击部署
- **THEN** 系统批量部署公钥到选中的通信机
- **THEN** 部署完成后弹出提示对话框，汇报部署成功的通信机数量以及失败的数量

### Requirement: 检查项管理
系统 SHALL 提供检查项的管理能力

#### Scenario: 检查项 CRUD
- **WHEN** 用户对检查项进行增删改查操作
- **THEN** 系统正确保存和返回检查项数据

#### Scenario: 检查项列表管理
- **WHEN** 用户管理检查项列表
- **THEN** 支持检查项列表的创建、修改、删除、复制、批量删除

#### Scenario: 检查项搜索
- **WHEN** 用户搜索检查项
- **THEN** 支持按名称搜索检查项列表

#### Scenario: 检查项类型
- **WHEN** 系统需要执行不同类型的检查
- **THEN** 支持以下检查类型：
  - 文件系统检查（磁盘空间、文件权限、修改时间、大小、MD5、配置文件比较、挂载检查）
  - 系统与进程检查（内核参数、路由表、服务状态、进程存在，黑名单）
  - 网络检查（端口监听状态）
  - 日志检查（日志文件扫描）
  - 文件内容检查（普通文本、配置文件、内核参数文件）

#### Scenario: 检查项菜单页面
- **WHEN** 用户从导航栏点击"检查项列表"或"检查项管理"
- **THEN** 进入检查项列表菜单页面

#### Scenario: 检查项列表管理界面
- **WHEN** 用户在检查项列表管理界面
- **THEN** 左侧展示所有检查项列表，提供新建、修改、克隆、删除按钮
- **THEN** 左侧包含"全部检查项"选项
- **THEN** 选择左侧所有检查项列表中的其中一个检查项列表时，右侧展示出该检查项列表下的所有检查项，提供新增、修改、克隆、删除功能
- **THEN** 点击左侧"全部检查项"选项时，右侧显示所有检查项列表包含的检查项
- **THEN** 右侧检查项表格显示所属检查项列表列

#### Scenario: 检查项对话框
- **WHEN** 用户点击新建/修改检查项列表
- **THEN** 弹出对话框要求输入检查项列表名称、描述等两项
- **WHEN** 用户点击新增检查项
- **THEN** 必须先选中一个检查项列表才能点击新增检查项按钮
- **THEN** 弹出对话框填写检查项信息，支持文件内容检查的详细配置
- **THEN** 对于文件/目录类型的检查项，支持多选检查类型（存在、权限、所有者、大小、修改时间、MD5码）
- **THEN** 对于每个选中的检查类型，支持配置比较基准（比较基准或指定值）
- **THEN** 对于指定值，支持设置具体的比对值和范围
- **THEN** 检查项将在选中的检查项列表中创建
- **THEN** 所有操作支持保存和取消功能

#### Scenario: 检查项约束
- **WHEN** 系统处理检查项
- **THEN** 检查项必须是在某个检查项列表下的，不能独立存在
- **THEN** 新建检查项时必须指定所属检查项列表

### Requirement: 快照管理
系统 SHALL 提供快照的创建和管理能力

#### Scenario: 快照 CRUD
- **WHEN** 用户对快照进行增删改查操作
- **THEN** 系统正确保存和返回快照数据

#### Scenario: 快照组管理
- **WHEN** 用户管理快照分组
- **THEN** 支持快照组的树形结构管理

#### Scenario: 快照类型
- **WHEN** 用户创建快照时
- **THEN** 支持以下快照类型：
  - 环境模板：只包含一个快照实例，作为标准参考
  - 环境快照：包含多个快照实例，代表多台通信机的环境状态
  - 缺省快照：每个快照组都有一个缺省快照

#### Scenario: 快照复制
- **WHEN** 用户复制快照
- **THEN** 支持快照复制功能

#### Scenario: 快照管理页面布局
- **WHEN** 用户访问快照管理页面
- **THEN** 页面显示左右布局
- **THEN** 左侧显示快照组管理，包含快照组树形结构
- **THEN** 左侧支持快照组的创建、编辑、删除操作
- **THEN** 右侧显示快照列表，显示当前选中快照组下的所有快照
- **THEN** 右侧支持快照的创建、查看、删除操作
- **THEN** 右侧支持按快照名称搜索
- **THEN** 点击左侧快照组时，右侧快照列表自动更新为该组下的快照

### Requirement: 检查规则管理
系统 SHALL 提供检查规则的管理能力

#### Scenario: 检查规则 CRUD
- **WHEN** 用户管理检查规则
- **THEN** 支持检查规则的创建、修改、删除、查看

#### Scenario: 检查规则配置
- **WHEN** 用户配置检查规则
- **THEN** 支持配置检查时间段、规则角色、关联的通信机/快照/检查项列表

### Requirement: 环境检查
系统 SHALL 提供环境检查的执行能力

#### Scenario: 手动启动检查
- **WHEN** 用户选择检查规则并手动启动检查
- **THEN** 系统异步执行检查任务，返回检查进度和结果

#### Scenario: 定时启动检查
- **WHEN** 用户配置定时任务并到达执行时间
- **THEN** 系统自动启动检查任务

#### Scenario: 快照不存在时比较
- **WHEN** 用户启动检查但基准快照不存在
- **THEN** 系统自动与缺省快照进行比较

#### Scenario: 通信机连接失败
- **WHEN** 检查过程中无法连接通信机
- **THEN** 系统记录检查失败并标记相应状态

### Requirement: 报表功能

#### Scenario: 查看检查报告

- **THEN** 系统展示包含成功/失败项统计的报表

#### Scenario: 导出报告
- **WHEN** 用户导出检查报告
- **THEN** 支持 PDF/Excel 格式导出

### Requirement: 历史数据管理

#### Scenario: 清理历史数据
- **WHEN** 用户配置数据保留策略
- **THEN** 系统按策略自动清理过期数据

#### Scenario: 查看历史结果
- **WHEN** 用户查看历史检查结果
- **THEN** 系统展示历史检查记录的详细信息

### Requirement: 配置管理
系统 SHALL 提供系统配置管理能力

#### Scenario: 配置导入导出
- **WHEN** 用户导入或导出系统配置
- **THEN** 系统正确处理配置数据的序列化

## 技术实现要求

### Requirement: 性能要求
- 操作员有效操作在 5 秒内响应，耗时操作提供进度显示
- 支持 5 个操作员并发访问
- 同一时间只能执行一项运行环境检查

### Requirement: 环境要求
- 支持 Linux 通信机

### Requirement: 部署要求
- 使用 Docker Compose 部署
- 支持 Nginx + Uvicorn 运行
- 使用 SQLite 开发 / PostgreSQL 生产

### Requirement: 用户界面要求
- 采用深交所官网设计风格
- 主色调：深交所红 (#C4181F)、深红 (#A0141A)、浅红 (#E6454C)
- 辅助色：深蓝 (#1E3A8A)、中蓝 (#3B82F6)、浅蓝 (#DBEAFE)
- 提供直观的检查规则配置界面
- 提供清晰的检查结果列表界面
- 提供详细的检查报告界面
- 提供通信机树形导航界面
- 支持按名称/IP 搜索通信机

### Requirement: 前端设计系统
- 字体：Microsoft YaHei (中文), Arial (英文), Consolas (代码)
- 间距系统：基础单位 4px，等级 xs/sm/md/lg/xl/2xl/3xl
- 按钮样式：主按钮 (深交所红), 次按钮 (白底红边), 文字按钮 (蓝色)
- 卡片样式：白色背景，圆角 8px，轻微阴影
- 表格样式：浅灰表头，斑马纹，悬停浅蓝背景
- 导航栏：深蓝背景，白色文字，悬停浅蓝，激活深红
- 状态指示器：绿 (成功)/黄 (警告)/红 (错误)/灰 (未知)
- 响应式布局：断点 640px/768px/1024px/1280px
