登录和通信机显示问题诊断

发现的问题：

1. 登录API请求格式不匹配

问题：前端使用 FormData 发送登录请求，但后端期望 OAuth2PasswordRequestForm 格式

当前代码：

// login.html
const formData = new FormData();
formData.append('username', username);
formData.append('password', password);

const response = await fetch('/api/v1/users/login', {
    method: 'POST',
    body: formData
});

修复方案：使用 URLSearchParams 而不是 FormData

const formData = new URLSearchParams();
formData.append('username', username);
formData.append('password', password);

const response = await fetch('/api/v1/users/login', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: formData
});

2. 通信机列表可能为空

可能原因：

  • 数据库中没有通信机数据
  • API认证失败
  • 前端JavaScript错误

调试步骤：

  1. 打开浏览器开发者工具 (F12)
  2. 查看 Console 标签页，检查是否有JavaScript错误
  3. 查看 Network 标签页，检查API请求是否成功
  4. 检查 localStorage 中是否存储了 token 和 username
  5. 在Console中运行：console.log('Token:', localStorage.getItem('token'))
  6. 在Console中运行：console.log('Username:', localStorage.getItem('username'))
